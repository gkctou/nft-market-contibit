EnftyMart

NFT Marketplace Vue JS Template.

EnftyMart is a Vue JS template which is based on latest Bootstrap 5 and its well optimized with mobile first responsive approach. EnftyMart is built using SASS preprocessor, HTML5, CSS3 and help of usefull npm plugins. The template is built with the latest Vue Version, making it completely free of jQuery dependency.

You can use this template for any kind of NFT Marketplace, digital marketplace, crypto collectibles and non-fungible tokens.

Quick Start - install

Download the latest theme source from themeforest.

Download and install Node.js from nodejs.org/en/download/ (if your computer does't have it)

  • How to run your vue project please follow the structure below:
  • Step 1 - go to /path-to-dir/enftymart-vue folder/directory
  • Step 2 - open the command prompt window or terminal in the enftymart-vue folder
  • Step 3 - Then run this command npm install in your enftymart-vue folder
  • Step 4 - After install then run this command npm run serve in your enftymart-vue folder
  • Step 5 - Open http://localhost:8080 to view it in the browser.

Production Build

  • - For production build, use npm run build. A build folder will be created in the root directory named dist
npm run build

Creating a Component

To create a .vue file in your project directory. you just right click on your project directory folder and name it with your own name like HelloWorld.vue. If you prefer terminal commands, you can do this in your computer’s terminal with the bash command touch: 

touch src/components/HellowWorld.vue

In this step, you are creating a single-file component that will contain all of the HTML, JavaScript, and css that this chunk of code needs. Every .vue component contains three basic concerns or sections: <template> <script> and <style> .In this case, template is the component’s HTML.

Open up the new file in your text editor.

In this file, create a div by using the <div> tag inside of <template> Inside of this <div> add the <h1> element with the content Hello World 

<template>
   <div class="hello">
        <h1>Hellow World<h1>
   <div>
</template>
<script>
export default {
    name: 'HelloWorld'
  }
</script>

Keep your development server running throughout development. If you close or cancel the server process, you will not be able to view your application in the browser.

Save the file.

At this point, when you open your browser, you will not see the HTML rendered. That is because you need to import the newly created HellowWorld.vue component into a component that is already mounted. Since App.vue is your main entry point, it’s best to import it to see the output

Open up the App.vue file in your text editor. and add the following code:

<template>
    <HellowWorld></HellowWorld> // <= 3 HellowWorld component imported
</template>
<script>
import HellowWorld from '@/components/HellowWorld.vue'// < 1 first import here then register its name in the component object 
export default {
    components: {
        HellowWorld // < 2 register here then
    }
 }
</script>

When you import using ES6, you are essentially creating a variable to later reference in your code. In this case, you are storing the code from HellowWorld.vue into a variable called HellowWorld You need to register it via the component property before you can use it.

After completing this step, save any unsaved file and open your browser back to localhost:8080 You will now find your newly created component at the top of the page:

Deploy in cPanel

To run your vue project in cPanel follow the instruction below

  • Step 1. Run npm run build command to your folder directory.
  • Step 2. create a file named .htaccess to your root directory where dist folder is. and include the below code to .htaccess file
  • <IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.html$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule . /index.html [L]
</IfModule>
  • Step 3. Upload your dist folder and .htaccess file to cPanel public_html directory ( or the directory of your sub domain, if its one)
